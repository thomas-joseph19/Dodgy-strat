# Quick Task 260427-o5n: Reset LiveEngine state on NT connection

## Context
When running NinjaTrader historical backtests multiple times, the Python server (`live_engine`) stays alive across connections. Because the internal state (e.g. `self._trade`, FVG history, swing history, `_last_date`) is not reset upon a new connection, subsequent backtests process historical bars with corrupted "future" state. This causes the strategy to take no trades on second and subsequent backtest runs.

## Task 1: Reset Engine State on Client Connection
- **Files**: `live/ninja_bridge.py`
- **Action**: In `_handle_client`, call `self.__init__(...)` to completely reset the underlying engine, processors, and safety guard state so that each new NinjaTrader connection starts with a fresh session. Re-assign `self._writer = writer` after initialization.
- **Verify**: The bridge successfully clears all prior history and takes trades correctly on consecutive historical backtest runs.
- **Done**: When `ninja_bridge.py` is updated.
